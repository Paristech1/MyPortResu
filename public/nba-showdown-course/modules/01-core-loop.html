The Core Loop

Imagine you're arguing with your friends about which NBA player had the best night. Instead of just looking at a boring spreadsheet, NBA Showdown turns it into a game. Two players appear on screen. You pick the best one. The winner stays; the loser goes home. You keep picking until only one Baller of the Night remains.

💡
The "Hot or Not" Pattern

This is a classic software pattern: taking a complex dataset (dozens of players with multiple stats) and reducing it to a series of simple, binary choices. It lowers the cognitive load for the user.

What happens when you click?

Every time you pick a player, the app runs a specific function to handle the match. Let's trace the journey of your click into the code.

CODE 
function handlePick(side) {
  const selected = side === 'left' ? leftPlayer : rightPlayer;
  const eliminated = side === 'left' ? rightPlayer : leftPlayer;

  if (deck.length < 1) {
    setWinner(selected);
    return;
  }

  const updatedDeck = [...deck, selected];
  const shuffledDeck = updatedDeck.sort(() => 0.5 - Math.random());
}
PLAIN ENGLISH

When a user clicks a card, start the match resolution.

Figure out who they picked based on the side they clicked.

Figure out who lost (the other guy).


If there's nobody left in the deck to play...

Crown the current selected player as the overall winner!

Stop here.


Otherwise, throw the winner back into the pool of players.

Shuffle the deck so the next match is random.

This creates an endless loop of matchups until the `deck` (our remaining player pool) is completely empty.
