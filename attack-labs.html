A hands-on lab working with the MITRE ATT&CK framework and the live ATT&CK Navigator — building technique layers and mapping adversary behavior to the tactics and techniques (TTPs) that defenders use to describe, hunt, and detect attacks. Lab Overview

MITRE ATT&CK is a globally-adopted knowledge base of real-world adversary behavior, organized into tactics (the attacker's goal — initial access, execution, persistence, privilege escalation, exfiltration) and the techniques they use to get there. The ATT&CK Navigator is the interactive matrix layered on top of it. This lab — completed through Sam Bowne's coursework (samsclass.info) — is hands-on practice driving the Navigator and thinking in TTPs.

  • The matrix: navigated the full ATT&CK enterprise matrix — tactics across the top, techniques and sub-techniques in each column.
  • Technique layers: built and colored Navigator layers to highlight and score techniques relevant to a scenario.
  • TTP mapping: practiced mapping observed behavior to a specific technique ID — the muscle a SOC analyst uses every shift.

Mapping Behavior to Techniques

The core skill ATT&CK builds is translating raw activity into a shared, searchable vocabulary. When a host runs something suspicious, the analyst's job is to name what the adversary did, not just that "something happened":

  • A cmd.exe command line maps to T1059.003 — Command and Scripting Interpreter: Windows Command Shell.
  • Each technique carries its tactic, data sources, detection guidance, and mitigations — so naming the TTP immediately points toward the telemetry and the detection that should fire.
  • Layering many mapped techniques in the Navigator turns a single incident into a coverage picture: what an adversary did, and where my detections have gaps.

Why It Matters

ATT&CK is the lingua franca of modern blue teams. Threat intel reports, detection rules, and SOC investigations all reference technique IDs, so fluency in the framework is foundational to detection engineering and incident response. This lab is the connective tissue between my other work — the adversary techniques I simulate in the AD attack lab, the Splunk detections I write to catch them, and the endpoint artifacts I collect in the Velociraptor DFIR lab all map back to ATT&CK.
